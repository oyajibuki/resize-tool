# GitHubアップロード用 コマンドチートシート

プロジェクト（フォルダ）をGitHubにアップロード（送信）する時によく使うコマンドのまとめです。
ターミナルやコマンドプロンプトにコピー＆ペーストして使えます。


## 1. 【初回】新しく作ったフォルダをGitHubにアップロードする時
新しいツールを作り、初めてGitHub上で公開する時に使います。
※ 事前にGitHubのサイト上で「New repository」から新しいリポジトリを作っておく必要があります。

```powershell
# ① まずはGitの管理対象として初期化する
git init

# ② フォルダ内のすべてのファイル（. は「すべて」という意味）をアップロード候補に追加する
git add .

# ③ 追加したファイルに「最初のコミットです」などの名前（メッセージ）を付ける
git commit -m "first commit"

# ④ アップロード先のURL（GitHubのリポジトリ）を設定する
# ※ 「リポジトリ名」の部分を、GitHubで作った名前に書き換えて実行してください。
git remote add origin https://github.com/oyajibuki/リポジトリ名.git

# ⑤ 現在の作業ブランチの名前を「main」に変更する
git branch -M main

# ⑥ GitHubへデータをアップロード（送信）する
# ※ 最初の1回だけ「-u origin main」を付けます。
git push -u origin main
```


---


## 2. 【2回目以降】コードを変更・更新してアップロードする時（よく使う！）
ツールを修正した時や、新しい機能を追加した時に「毎回使う」コマンドです。
この3つのコマンドさえ覚えておけば、今後もスムーズに作業できます！

```powershell
# ① 変更したすべてのファイルをアップロード候補に追加する
git add .

# ② 変更内容にメモ（メッセージ）を付けて保存する
# ※ メッセージの内容は「バグ修正」「機能追加」など、自分で分かりやすいものでOKです。
git commit -m "アップデート"

# ③ GitHubにアップロード（送信）する
# ※ 初回に設定を済ませているので、以降はこれだけでOK！
git push
```
